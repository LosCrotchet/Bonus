# Bonus 游戏项目架构文档

## 项目概述

**Bonus** 是一个基于 Godot 4.6 引擎开发的多人在线扑克牌游戏，融合了斗地主和骰子的玩法机制。游戏支持 2-4 名玩家，包含单人游戏和多人在线对战模式。

## 核心架构设计

### 1. 系统架构层次

#### 1.1 网络层 (WebController)
- **文件**: `system/web_controller/web_controller.gd`
- **职责**: 管理多人游戏网络连接、RPC通信、玩家状态同步
- **关键特性**:
  - 支持服务器/客户端模式
  - 基于 ENetMultiplayerPeer 的可靠网络通信
  - 自动玩家编号和房间管理
  - 实时状态同步机制

#### 1.2 游戏管理层 (DeckManager)
- **文件**: `system/deck_manager/deck_manager.gd`
- **职责**: 核心游戏逻辑、牌局管理、规则验证
- **关键特性**:
  - 卡牌类型识别和比较算法
  - 万能牌（Joker）逻辑处理
  - 牌型验证和大小比较
  - 游戏信号广播机制

#### 1.3 场景管理层 (GameManager)
- **文件**: `system/game_manager/game_manager.gd`
- **职责**: 回合管理、游戏流程控制、音效管理
- **关键特性**:
  - 回合制游戏流程控制
  - 庄家轮换机制
  - 音效播放系统
  - 游戏状态同步

### 2. 场景架构

#### 2.1 主菜单场景 (Menu)
- **文件**: `scene/menu/menu.tscn` + `scene/menu/menu.gd`
- **架构特点**:
  - 状态机管理模式 (6种界面状态)
  - 平滑的界面过渡动画系统
  - 玩家设置管理（名称、头像、IP地址）
  - 单人和多人游戏模式选择

#### 2.2 游戏场景 (GameScene)
- **文件**: `scene/game_scene/game_scene.tscn` + `scene/game_scene/game_scene.gd`
- **架构特点**:
  - 多玩家UI布局管理
  - 游戏结束处理流程
  - 场景切换和重启机制
  - 实时UI更新系统

### 3. 实体架构

#### 3.1 玩家系统 (Player System)

**玩家管理器 (PlayerManager)**
- **文件**: `entities/player/player_manager.gd`
- **职责**: 管理所有玩家实例、协调玩家行为
- **关键功能**:
  - 玩家位置布局（支持2-4人）
  - AI玩家决策系统
  - 玩家状态同步
  - 游戏结束判定

**玩家实体 (Player)**
- **文件**: `entities/player/player.gd`
- **职责**: 单个玩家的状态和行为管理
- **关键功能**:
  - 手牌管理（排序、选择、出牌）
  - 卡牌动画系统
  - 玩家UI交互
  - AI决策算法
  - 音效反馈系统

#### 3.2 卡牌系统 (Card System)
- **文件**: `entities/card/card.gd`
- **架构特点**:
  - 基于区域的卡牌显示系统
  - 卡牌状态管理（正反面、选中状态）
  - 花色和点数编码系统
  - 灵活的卡牌实例化机制

#### 3.3 骰子系统 (Dice System)
- **文件**: `entities/dice/dice.gd`
- **架构特点**:
  - 动画驱动的结果显示
  - 随机数生成和同步
  - 特殊状态显示（Bonus状态）
  - 网络同步机制

#### 3.4 波比角色 (Bobo)
- **文件**: `entities/bobo/bobo.gd`
- **职责**: 游戏中的角色互动元素

### 4. 数据流架构

#### 4.1 游戏状态流转
```
游戏开始 → 发牌 → 投掷骰子 → 出牌阶段 → 判定结果 → 回合切换 → 游戏结束
```

#### 4.2 网络通信流程
```
客户端操作 → WebController RPC → 服务器验证 → 状态广播 → 客户端更新
```

#### 4.3 事件驱动架构
- **信号系统**: 基于 Godot 的信号机制实现松耦合
- **关键信号**:
  - `GameSignal`: 游戏状态更新
  - `PlayerFinish`: 玩家操作完成
  - `multi_load`: 多人游戏加载同步
  - `dice_roll`: 骰子投掷事件

### 5. 核心算法设计

#### 5.1 牌型识别算法
- **递归算法**: 处理万能牌的组合可能性
- **排序算法**: 自定义卡牌比较规则
- **牌型分类**: 6大类23小类牌型识别

#### 5.2 AI决策系统
- **概率计算**: 基于手牌分析的出牌概率
- **策略选择**: 多种出牌策略的加权选择
- **难度适应**: 可调整的智能水平

#### 5.3 网络同步算法
- **状态压缩**: 高效的数据序列化
- **冲突解决**: 服务器权威模式
- **延迟补偿**: 客户端预测和服务器验证

### 6. UI/UX 架构

#### 6.1 界面布局系统
- **自适应布局**: 支持不同玩家数量的界面调整
- **位置管理**: 4个固定玩家位置（上、下、左、右）
- **状态显示**: 实时游戏信息反馈

#### 6.2 动画系统
- **Tween动画**: 平滑的界面过渡效果
- **卡牌动画**: 发牌、出牌、选择动画
- **特效系统**: 游戏事件视觉反馈

#### 6.3 音效系统
- **分层音效**: 按钮音、卡牌音、特效音
- **异步播放**: 不阻塞游戏流程的音效管理
- **音效选择**: 随机化音效增加变化

### 7. 技术特性

#### 7.1 多人游戏架构
- **P2P网络模型**: 基于ENet的高效通信
- **权威服务器**: 防止作弊的游戏逻辑
- **状态同步**: 实时游戏状态广播
- **断线重连**: 网络异常处理机制

#### 7.2 性能优化
- **对象池**: 卡牌实例的复用管理
- **异步加载**: 避免界面卡顿
- **内存管理**: 及时清理临时对象
- **网络优化**: 数据压缩和批量传输

#### 7.3 扩展性设计
- **模块化架构**: 各系统松耦合设计
- **配置驱动**: 游戏参数外部配置
- **插件系统**: 易于添加新功能

### 9. 关键技术栈

- **游戏引擎**: Godot 4.6 (GL Compatibility模式)
- **编程语言**: GDScript
- **网络库**: ENetMultiplayerPeer
- **渲染模式**: Canvas Items 2D
- **音频系统**: AudioStreamPlayer2D

## 总结

Bonus项目采用现代化的游戏架构设计，具有以下特点：

1. **清晰的层次分离**: 网络层、游戏逻辑层、表现层完全分离
2. **事件驱动设计**: 基于信号的系统间通信，降低耦合度
3. **可扩展架构**: 模块化设计便于功能扩展
4. **性能优化**: 多层面的性能优化策略
5. **用户体验**: 流畅的动画和音效设计

该架构支持从单人游戏到多人在线的完整游戏体验，具有良好的维护性和扩展性。
